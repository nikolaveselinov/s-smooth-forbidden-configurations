# Unit corners in smooth semigroups

This repository accompanies the paper []().

It contains exact computation code for the density intervals discussed in the paper and machine-readable output files for the examples computed by the author so far.

## Repository layout

- `scripts/`
  - `compute_density_intervals.py` — exact prefix optimization and density-interval computation.
- `data/`
  - `alpha23_200.json` — data for `S={2,3}`, `K=200`.
  - `alpha25_180.json` — data for `S={2,5}`, `K=180`.
  - `alpha35_180.json` — data for `S={3,5}`, `K=180`.
  - `alpha235_120.json` — data for `S={2,3,5}`, `K=120`.
  - `alpha2357_80.json` — data for `S={2,3,5,7}`, `K=80`.
  - `all_results.json` — combined output for all of the above runs.

## Python requirements

The script uses:

- Python 3.11+
- `numpy`
- `scipy`

## Reproducing the calculations

From the repository root, run for example:

```bash
python scripts/compute_density_intervals.py \
  --spec 2,3:200 \
  --digits 24 \
  --out data/alpha23_200_reproduced.json
```

Multiple prime sets can be computed in one run:

```bash
python scripts/compute_density_intervals.py \
  --spec 2,5:180 \
  --spec 3,5:180 \
  --digits 24 \
  --out data/two_prime_examples.json
```

The option format is

- `p1,p2,...:K`

so, for example, `2,3,5:120` means the first `120` numbers in the semigroup generated by `2,3,5`.

## Design of the script

For each finite set of primes `S` and each cutoff `K`, the script:

1. generates the first `K` positive `S`-smooth numbers;
2. solves the extremal problems `f_S(k)` for `1 <= k <= K`;
3. forms the increments `Delta f_S(k)`;
4. computes the lower and upper bounds for
   `alpha_S = c_S * sum_k Delta f_S(k) / d_k^(S)`
   using the tail formula.

## JSON output files

Each JSON output records:

- the prime set and cutoff;
- the last prefix optimum;
- the last smooth number;
- decimal lower and upper bounds for the density constant;
- the interval width;
- the full list of prefix optima;
- the full smooth-number prefix;
- the increment sequence.

The output is sufficient to verify the quoted numerical intervals and to continue the calculations at larger cutoffs.
